## 3. 每日计划

### Day 1：第 1 周：T1 稳定性

目标能力：T1

为什么安排：T1 当前 72.97，实现风险较高，必须先降低会做但错细节的问题。 本日属于 30 天计划第 1 周。

任务：
- D1-D1-1 [new_problem] implementation / 普及/提高-
  - 选题标准：选择一道模拟、枚举、前缀和或简单贪心题，重点训练边界和实现稳定性。
  - 训练目标：提高一次 AC 率，减少边界错误。
  - 限时：40 分钟
- D1-D1-2 [new_problem] prefix_sum_or_greedy / 普及+/提高-
  - 选题标准：选择一道需要处理边界或排序细节的题，提交前必须完成检查清单。
  - 训练目标：训练数据范围、下标、初始化检查。
  - 限时：45 分钟
- D1-D1-3 [old_problem_review] implementation / 历史旧题
  - 选题标准：从高价值复盘池选 1 道带 ARRAY_INDEX_RISK 或 DEFINE_INT_LONG_LONG 的旧题重看。
  - 训练目标：找出历史低级错误模式。
  - 限时：35 分钟

旧题复盘池：
- U465842：提交次数高，适合比较早期版本与最终版本。
- U519362：长期未 AC 或高提交次数，是卡题复盘优先样本。

达标标准：
- 新题提交次数 <= 2
- 提交前完成边界检查清单
- 记录错误类型

失败调整：
- 若 WA >= 2，Day2 前加 1 道同类基础题重做。

### Day 2：第 1 周：DP / 区间 DP

目标能力：T2

为什么安排：DP 覆盖较多但平均提交次数偏高；区间 DP 是高优先级模块，需要稳定状态设计。 本日属于 30 天计划第 1 周。

任务：
- D2-D2-1 [new_problem] linear_dp / 提高-
  - 选题标准：选择一道线性 DP，写清状态、转移、初始化、遍历顺序。
  - 训练目标：把题意稳定转为状态。
  - 限时：50 分钟
- D2-D2-2 [new_problem] knapsack_dp / 提高-
  - 选题标准：选择一道背包或计数 DP，重点检查容量循环方向和初始化。
  - 训练目标：减少 DP 边界与循环顺序错误。
  - 限时：55 分钟
- D2-D2-3 [new_problem] interval_dp / 提高/提高+
  - 选题标准：选择一道区间 DP 中档题，必须写 len 枚举与转移依据。
  - 训练目标：专项补强区间 DP。
  - 限时：70 分钟

旧题复盘池：
- P8699：提交次数高，适合比较早期版本与最终版本。

达标标准：
- 每题写状态定义
- 每题写复杂度
- 至少 1 题独立 AC

失败调整：
- 若状态设计不清，Day4 复盘加入区间 DP 旧题。

### Day 3：第 1 周：图论 / 并查集 / 二分

目标能力：T2

为什么安排：并查集和二分平均提交次数偏高，需要训练集合含义、check 单调性和图建模稳定性。 本日属于 30 天计划第 1 周。

任务：
- D3-D3-1 [new_problem] dsu / 提高-
  - 选题标准：选择一道并查集题，写清每个集合表示什么、合并条件是什么。
  - 训练目标：降低并查集建模和实现错误。
  - 限时：45 分钟
- D3-D3-2 [new_problem] binary_search / 提高
  - 选题标准：选择一道二分答案题，先证明 check 单调性再写代码。
  - 训练目标：稳定二分边界和答案定义。
  - 限时：60 分钟
- D3-D3-3 [new_problem] graph / 提高
  - 选题标准：选择一道图遍历/最短路建模题，写清点、边、权和复杂度。
  - 训练目标：提升图论建模转化。
  - 限时：60 分钟

旧题复盘池：
- P1955：提交次数高，适合比较早期版本与最终版本。
- P6691：提交次数高，适合比较早期版本与最终版本。

达标标准：
- 二分题 check 单调性写清楚
- 并查集题集合含义写清楚
- 图论题复杂度写清楚

失败调整：
- 若 TLE 或复杂度错误，Day4 复盘必须加入复杂度分析。

- P3585：提交次数高，适合比较早期版本与最终版本。
- P1955：提交次数高，适合比较早期版本与最终版本。
- P1075：提交次数高，适合比较早期版本与最终版本。

达标标准：
