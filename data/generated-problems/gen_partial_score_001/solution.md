## 部分分解法

### 30 分暴力做法
由于 $N$ 很小，我们可以枚举所有可能的分割点组合。在 $N-1$ 个可能的切割位置中选择 $M-1$ 个，不同的组合数最多为 $C_9^4 = 126$，对于每个组合计算代价，取最小值。计算代价需要遍历每个子段求最大值，总复杂度 $O(C_{N-1}^{M-1} \times N)$，可轻松通过 30% 的数据。

### 70 分动态规划
设 $\mathrm{dp}[i][j]$ 表示前 $i$ 个数分成 $j$ 段的最小代价。转移方程：
$$\mathrm{dp}[i][j] = \min_{0 \le k < i} \left( \mathrm{dp}[k][j-1] + \max_{t=k+1}^{i} A_t \right)$$
初始：$\mathrm{dp}[0][0] = 0$，其余 $\mathrm{dp}[i][j] = \infty$（$i<j$ 时无意义）。
直接枚举 $i,j,k$ 并计算区间最大值，复杂度 $O(N^2 M)$，可应对 $N \le 200,\, M \le 20$。

### 100 分单调栈优化 DP
观察 70 分 DP 的转移，当 $j$ 固定时，数组 $g[i] = \mathrm{dp}[i][j]$ 只依赖于 $f[i] = \mathrm{dp}[i][j-1]$。转移形式为：
$$g[i] = \min_{0 \le k < i} \left( f[k] + \max_{t=k+1}^{i} A_t \right)$$
这个子问题和“将数组任意分成若干段使得每段最大值之和最小”的 DP 完全一致，只是初始的 $f$ 数组不同。该问题可以用单调栈在 $O(N)$ 时间内求解。

具体做法：维护一个栈，栈中元素是 $(pos, val)$ 对，其中 $pos$ 是某个位置的下标，$val$ 表示以 $pos$ 为上一段的结束点时，当前段（即从 $pos+1$ 到当前位置）的代价候选值，即 $f[pos] + \max_{pos+1..current} A$。栈内元素满足 $A[pos]$ 递减（或严格地，$A[pos]$ 不小于栈内更深位置的 $A$）且 $val$ 也是一个非降序列。

处理到 $A[i]$ 时：
- 先以单独把 $A[i]$ 作为一个新段来获得一个候选值 $cur = f[i-1] + A[i]$（如果 $f[i-1]=\infty$ 则 $cur=\infty$）。
- 不断弹出栈顶元素 $(p, v)$，直到栈空或 $A[p] > A[i]$。对于弹出的每个 $(p, v)$，因为 $A[i] \ge A[p]$，原先以 $p$ 为分界点的那一段的最大值会从 $A[p]$ 变成 $A[i]$，从而新的候选值为 $v - A[p] + A[i]$。用这个值更新 $cur$。
- 如果栈未空，则栈顶元素的 $v$ 本身也是一个候选（因为其对应的段的最大值不变），用其更新 $cur$。
- 将 $(i, cur)$ 压入栈中，并令 $g[i] = cur$。

这样，每个位置入栈出栈各一次，总时间 $O(N)$。于是整个 DP 的时间复杂度降为 $O(NM)$，可以通过全部数据。
