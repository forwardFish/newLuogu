# 彩色路径 题解

## 解法概述

本题是一个在有向无环图（DAG）上的动态规划问题，关键在于正确设置状态并进行转移。由于答案需要对 $10^9+7$ 取模，注意使用长整型中间存储。

## 状态定义

设 $dp[u][0]$ 表示从起点 $1$ 到节点 $u$，且路径以蓝色节点 $u$ 结束的合法路径数；$dp[u][1]$ 表示以红色节点 $u$ 结束的合法路径数。特别地，长度为 $0$ 的路径（仅包含节点 $1$）也算作以 $1$ 的自身颜色结束。

## 初始化

起点 $1$：
- 若 $c_1 = 0$（蓝色），则 $dp[1][0] = 1$，$dp[1][1] = 0$。
- 若 $c_1 = 1$（红色），则 $dp[1][1] = 1$，$dp[1][0] = 0$。

其他节点初始均为 $0$。

## 状态转移

按照拓扑序依次处理每个节点 $u$。对于从 $u$ 出发的每条边 $(u, v)$，转移规则如下：

- 如果节点 $v$ 是蓝色（$c_v = 0$），则无论 $u$ 是什么颜色，都可以从 $u$ 走到 $v$，因此：
  $$ dp[v][0] \leftarrow dp[v][0] + dp[u][0] + dp[u][1] $$
- 如果节点 $v$ 是红色（$c_v = 1$），则只有当 $u$ 是蓝色时才能走到 $v$（红色不能接红色），因此：
  $$ dp[v][1] \leftarrow dp[v][1] + dp[u][0] $$

上述加法均在模 $10^9+7$ 下进行。

## 边界与答案

最终答案为 $dp[N][0] + dp[N][1] \pmod{10^9+7}$。

## 拓扑排序

对于子任务 2，由于题目保证所有边满足 $u < v$，可以直接按节点编号从小到大的顺序进行 DP，天然满足拓扑序。

对于子任务 3，需要显式求出图的拓扑序。常用方法是 Kahn 算法（基于入度）或深度优先搜索（DFS）后序遍历的逆序。以下标准算法使用 Kahn 算法：

- 计算每个节点的入度。
- 将所有入度为 $0$ 的节点加入队列（注意：起点 $1$ 可能入度不为 $0$，但我们必须从入度为 $0$ 的节点开始处理，因为 DAG 保证至少有一个入度为 $0$ 的节点；但 $1$ 不一定入度为零，然而我们仍可以处理，因为 DP 只会从已计算的前驱传播）。实际上，Kahn 算法会按拓扑序依次处理所有节点，对于任意边 $u \to v$，$u$ 会在 $v$ 之前处理。因此这样转移是正确的。

## 时间复杂度

拓扑排序和 DP 均只遍历每条边一次，总时间复杂度为 $O(N + M)$，空间复杂度 $O(N + M)$，可以在 $2\times 10^5$ 的规模下高效运行。

## 子任务得分策略

- 子任务 1 可以直接用深度优先搜索（DFS）枚举所有路径，在路径末尾判断合法性并计数。
- 子任务 2 可利用 $u<v$ 的性质，按节点编号顺序进行 DP。
- 子任务 3 需要完整的拓扑排序 DP。
