# 测试数据说明

测试数据使用 `generator` 程序生成，参数为 `subtask N M seed`。

- **子任务 1 (30分)**：共 5 组测试点。N ≤ 10，M ≤ 45，答案较小或中等。部分测试点颜色全为蓝色，部分混合。所有测试点均保证存在从 1 到 N 的路径。
- **子任务 2 (50分)**：共 5 组测试点。N ≤ 1000，M ≤ 5000，且输入满足所有边 u < v，即节点编号本身就构成拓扑序，无需进行拓扑排序。
- **子任务 3 (100分)**：共 10 组测试点。N ≤ 100,000，M ≤ 200,000，图是任意的 DAG，且节点编号不一定为拓扑序。需要选手自己实现拓扑排序。

所有测试点的颜色随机 (0/1)，且保证至少存在一条从 1 到 N 的路径（直接添加了一条边 1 → N）。
测试点根据种子随机生成，确保覆盖各种边界情况（包括 N=1 的情况）。
