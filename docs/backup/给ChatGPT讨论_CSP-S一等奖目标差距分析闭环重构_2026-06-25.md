## 2. 当前报告为什么还不能叫“CSP-S 一等奖差距分析报告”

### 2.1 没有真实的 CSP-S 历年真题基准库

当前文件：

`data/problem-bank/csp_s_problem_map.json`

名字看起来像 CSP-S 真题映射，但实际内容主要是把我的历史洛谷题目粗略放进：

- `T1_STABILITY`
- `T2_MODELING`
- `T3_PARTIAL_SCORE`
- `T4_STRATEGY`

当前数量为：

```text
T1_STABILITY：0
T2_MODELING：100
T3_PARTIAL_SCORE：80
T4_STRATEGY：80
```

这暴露了两个问题：

1. 它不是 CSP-S 2019—2025 历年真题数据库；
2. T1 为空，说明连四题位置能力映射都不完整。

因此它不能回答：

- 近 7 年 CSP-S T1 常考什么？
- 我对这些 T1 能力覆盖了多少？
- 哪些历年真题我做过？
- 哪些真题能力我完全没有证据？
- 我的薄弱点通常影响 T1、T2、T3 还是 T4？

### 2.2 T1/T2/T3/T4 当前只是抽象能力标签

现有系统把：

- T1 理解为稳定性；
- T2 理解为建模；
- T3 理解为部分分；
- T4 理解为综合策略。

这种抽象有一定训练价值，但不能替代真实真题位置分析。

原因是：

1. 某年的 T1 也可能有明显算法要求；
2. 某年的 T2 不一定只考建模；
3. T3/T4 的部分分结构每年不同；
4. 真题位置、知识点、难度和可拿分结构必须分别保存；
5. 不能因为一道普通洛谷题需要建模，就自动认为它等价于 CSP-S T2。

因此，T1/T2/T3/T4 应该同时拥有两种含义：

```text
A. 真题位置：当年四道题中的第几题
B. 能力角色：稳定拿分、建模转化、部分分、综合策略
```

二者有关联，但不能混为一谈。

### 2.3 算法模块识别存在误判

当前系统把 54 道历史题关联到区间 DP。

但是根据已有验证：

- 区间 DP 薄弱点的模拟验证为 0/3 命中；
- 部分证据题并非典型区间 DP；
- 静态规则可能把二维数组、区间循环或类似转移误判为区间 DP。

所以“区间 DP 是我的主要短板”目前不能被视为确定结论。

它应当降级为：

> 待复核的算法识别假设。

系统需要区分：

```text
检测到疑似区间结构
检测到区间 DP 模板
题目本身属于区间 DP
错误根因确实是区间 DP 状态或枚举顺序
```

这四件事不是同一件事。

### 2.4 “数学建模不足”无法直接指导训练

当前大量题被归为 `math`，并进一步汇总成“数学建模不足”。

但是“数学”至少需要拆成：

- 数论基础；
- 整除、同余与模运算；
- 最大公约数、质因数分解；
- 组合计数；
- 容斥与计数模型；
- 递推关系；
- 数学式推导；
- 数学式转算法；
- 大整数和溢出；
- 浮点与精度；
- 数学模型与 DP 模型的边界；
- 构造题中的数学性质；
- 周期、奇偶和不变量。

如果不细分，就无法判断：

- 我到底是不会数论，还是不会从题意推公式？
- 是模型错误，还是代码溢出？
- 是复杂度判断错误，还是部分分策略不足？
- 应该补知识，还是应该做建模迁移训练？

### 2.5 没有建立真题知识点频率

系统目前没有形成：

```text
2019—2025 CSP-S 真题
× T1/T2/T3/T4 位置
× 一级知识点
× 二级知识点
× 三级能力
× 出现频率
× 分值影响
```

因此当前严重度只说明“历史上看起来问题较多”，并不说明：

> 这个问题对 CSP-S 一等奖到底有多重要。

例如：

- 一个历史上高频出现、但 CSP-S 很少考的细节，不应该排在第一优先级；
- 一个历史样本少、但会导致 T1 爆零的错误，可能应当非常高优先级；
- T3/T4 的部分分能力虽然不属于单一知识点，却可能直接决定一等奖。

### 2.6 没有建立预计失分模型

当前系统没有回答：

- 如果 T1 边界问题不修复，典型套题可能损失多少分？
- 如果不会二分答案，近年真题中会影响哪一题？
- 如果部分分拆解不足，T3/T4 通常会损失多少可获得分数？
- 如果区间 DP 判断是误报，继续投入训练会浪费多少时间？
- 哪个弱点最可能让我低于目标省一等奖线？

没有预计失分，就无法科学排序训练优先级。

### 2.7 没有目标省份的一等奖线模型

CSP-S 一等奖线不是固定值。

官方公开信息显示：

- 2024 年提高级一等全国认证基准分数线为 165 分；
- 2025 年提高组一级全国认证基准分数线为 131 分；
- 各省实际分数线存在明显差异；
- 同一省份不同年份也可能明显波动。

官方来源：

- 2024 评级规则：<https://www.noi.cn/xw/2024-11-19/834421.shtml>
- 2025 评级规则：<https://www.noi.cn/xw/2025-11-14/853110.shtml>

因此系统必须知道：

```text
目标省份
目标年份
近 3～5 年该省一等奖线
历史最低线
历史中位线
历史最高线
年度波动
安全分数垫
```

在不知道目标省份、也没有真实套题成绩时，不能给出精确的一等奖概率。

## 3. 正确的 CSP-S 一等奖分析应该先建立三个基准库

## 3.1 基准库一：CSP-S 2019—2025 历年真题库

至少应覆盖 2019—2025 每年的四道提高级第二轮题目。

每道题不能只保存题号和算法标签，而应拥有以下字段：

```json
{
  "year": 2024,
  "problemId": "真实题号",
  "title": "题目名称",
  "slot": "T1",
  "officialFullScore": 100,
  "primaryKnowledge": [],
  "secondaryKnowledge": [],
  "knowledgePath": [],
  "modelType": [],
  "requiredTransformations": [],
  "implementationRisks": [],
  "partialScoreStructure": [],
  "expectedComplexity": "",
  "fullSolutionDifficulty": "",
  "firstPrizeExpectedScore": {
    "conservative": 0,
    "normal": 0,
    "strong": 0
  },
  "commonFailureModes": [],
  "sourceUrls": [],
  "classificationConfidence": 0
}
```

每道题需要回答：

1. 它位于 T1/T2/T3/T4 的哪个位置？
2. 主知识点是什么？
3. 次知识点是什么？
4. 真正难点是知识、建模、实现还是策略？
5. 题意到算法需要哪些关键转化？
6. 满分算法是什么？
7. 有哪些可拿部分分？
8. 暴力能拿多少？
9. 特殊性质能拿多少？
10. 一等奖目标选手合理目标分是多少？
11. 常见错误是什么？
12. 是否适合用作某项弱点的验证题？

建议把一道题拆成以下维度：

```text
题目位置
→ 考场角色
→ 知识模块
→ 模型类型
→ 关键转化
→ 实现风险
→ 部分分结构
→ 预计得分
```

例：

```text
主模块：动态规划
二级知识点：区间 DP
三级能力：状态含义、转移推导、初始化、枚举顺序
关键转化：把操作过程转成区间合并状态
实现风险：len 枚举、左右端点、空区间、初始值
部分分：小规模搜索、特殊结构、低维 DP、完整 DP
考场角色：T2 满分或 T3 部分分
```

参考入口：

- 洛谷 CSP-S 历年真题题单：<https://www.luogu.com.cn/training/551859>
- NOI 官网历年资料：<https://www.noi.cn/>

需要注意：

> 洛谷题单可以作为题目索引，但题目分类和能力解释必须由系统重新审核，不能直接把社区标签当作最终真值。

## 3.2 基准库二：CSP-S 知识能力树

知识树应参考 NOI 大纲，而不是只依靠代码关键字。

NOI 官网已经发布 2025 年修订版大纲：

- 发布说明：<https://www.noi.cn/xw/2025-04-18/841584.shtml>
- NOI 官网：<https://www.noi.cn/>

建议建立三级甚至四级能力树。

### 第一层：知识领域

- 程序设计基础；
- 基础算法；
- 数据结构；
- 图论；
- 动态规划；
- 数学；
- 字符串；
- 搜索；
- 计算几何；
- 综合算法。

### 第二层：算法模块

以动态规划为例：

- 线性 DP；
- 背包 DP；
- 区间 DP；
- 树形 DP；
- 状压 DP；
- 数位 DP；
- DAG DP；
- 计数 DP；
- 优化 DP。

### 第三层：可观察能力

以区间 DP 为例：

- 识别区间子结构；
- 定义 `dp[l][r]` 的含义；
- 确定空区间或单点初始化；
- 推导断点转移；
- 确定区间长度枚举顺序；
- 处理环形区间；
- 判断复杂度；
- 设计部分分。

### 第四层：错误模式

- 状态语义错误；
- 状态维度不足；
- 初始化错误；
- 转移漏项；
- 枚举顺序错误；
- 左右端点越界；
- INF 或负无穷设置错误；
- 空间或时间复杂度超限。

### 非知识点能力树

CSP-S 一等奖不能只看知识点，还应单独建模：

#### 读题能力

- 提取输入规模；
- 提取特殊性质；
- 区分必要条件和充分条件；
- 识别隐含限制；
- 把自然语言变成数学对象。

#### 建模能力

- 变量定义；
- 状态定义；
- 图模型定义；
- 集合含义；
- 单调性证明；
- 不变量识别；
- 问题等价转化。

#### 复杂度能力

- 根据数据范围反推算法；
- 估算常数；
- 判断空间限制；
- 选择暴力、优化或正解。

#### 实现稳定性

- 数组边界；
- 初始化；
- 数据类型；
- 多测清空；
- 递归栈；
- 输入输出；
- 模板正确性；
- 样例之外的自测。

#### 部分分能力

- 识别小规模子任务；
- 暴力搜索；
- 特殊性质；
- 降维或降复杂度；
- 从 30 分逐步提升到 50/70 分；
- 保证部分分代码正确。

#### 调试与验证能力

- 设计边界样例；
- 设计随机数据；
- 暴力对拍；
- 定位错误阶段；
- 判断继续调试还是换题。

#### 考场策略

- 开题顺序；
- 时间预算；
- 止损；
- 回看；
- T1/T2 保分；
- T3/T4 部分分；
- 最后检查；
- 文件名、编译、Linux 环境稳定性。

## 3.3 基准库三：一等奖得分与风险模型

系统不能只输出一个 readiness 数字。

建议保存：

```json
{
  "province": "待填写",
  "targetYear": 2026,
  "historicalLines": [
    {
      "year": 2024,
      "nationalBaseline": 165,
      "provinceLine": null,
      "source": "official"
    },
    {
      "year": 2025,
      "nationalBaseline": 131,
      "provinceLine": null,
      "source": "official"
    }
  ],
  "statistics": {
    "min": null,
    "median": null,
    "max": null,
    "volatility": null
  },
  "safetyMargin": 20,
  "targetSafeScore": null
}
```

建议至少输出三种目标线：

```text
最低过线目标：参考近年较低分数线
稳健目标：近年中位线 + 10～20 分
高安全目标：近年高位线 + 20 分
```

同时，系统必须输出不确定性：

- 省份未知；
- 分数线波动；
- 真题难度变化；
- 数据样本不足；
- 模拟成绩不等于真实成绩；
- 日常题目成绩不等于四小时套题成绩。

## 4. “薄弱点命中”必须经过五层证据链

当前系统不能因为静态规则检测到某种模式，就直接宣布这是主要薄弱点。

每个薄弱点必须经过以下五层。

## 4.1 第一层：历史证据命中

需要证明历史上反复发生：

- 多次 WA；
- 部分分长期停滞；
- 多次提交后才 AC；
- 长期未 AC；
- 版本修改集中在相同位置；
- 同模块不同题重复失败；
- 错误在近期仍出现；
- 不是只由一道题造成。

建议最低证据标准：

```text
至少 3 道不同题
或者至少 2 道题且具有清晰版本修复证据
或者 1 道高价值真题且有完整思考/调试记录
```

单题样本不能直接形成高置信度模块结论。

## 4.2 第二层：知识点命中

薄弱点必须定位到三级知识点或非知识点能力。

不合格：

```text
DP 不好
数学不好
图论建模不足
实现有风险
```

合格：

```text
动态规划
→ 区间 DP
→ 状态含义和 len 枚举顺序
→ 在两道新题中重复出现初始化遗漏
```

或者：

```text
数学建模
→ 模运算与周期
→ 没有从数据范围识别周期性质
→ 导致只写出超时暴力
```

## 4.3 第三层：CSP-S 真题命中

系统需要统计：

- 2019—2025 出现次数；
- 出现年份；
- 常见题目位置；
- 是满分知识还是部分分知识；
- 对一等奖得分的重要性；
- 对应哪些真实真题。

例如输出：

```text
薄弱点：二分答案的 check 单调性证明
近 7 年相关真题：X 道
常见位置：T1/T2
合理目标：相关题应争取 70～100 分
当前掌握：待验证
预计风险：若误判单调性，可能整题失分
```

## 4.4 第四层：预计失分命中

每个弱点必须换算成考场风险。

建议公式：

```text
预计失分风险
= 真题出现概率
× 该知识点在题目中的分值权重
× 当前失败概率
× 无替代部分分的风险
× 一等奖线敏感度
```

可以拆成：

```text
frequencyScore：近年出现频率
positionWeight：T1/T2/T3/T4 位置权重
failureProbability：根据历史和盲测估计
recoverableScore：失败后仍可拿的部分分
scoreLossExpected：预计失分
```

输出示例：

```json
{
  "weakness": "T1 边界检查不稳定",
  "appearanceProbability": 0.8,
  "failureProbability": 0.25,
  "typicalFullScore": 100,
  "recoverableScore": 20,
  "expectedScoreLoss": 16,
  "firstPrizeImpact": "HIGH"
}
```

这个数值不需要假装绝对精确，但必须说明依据和置信区间。

## 4.5 第五层：新题盲测命中

最终必须用未做过的题验证。

验证流程：

1. 系统在做题前预测薄弱点；
2. 选择未做过的 CSP-S 真题、同类原题、变式题和迁移题；
3. 我不看题解完成；
4. 记录七个阶段；
5. 对比真实错误是否命中预测；
6. 更新置信度和优先级。

七阶段建议为：

```text
reading：读题和约束提取
modeling：模型转化
partialScorePlanning：部分分规划
algorithmDesign：算法设计
implementation：代码实现
debugging：调试
review：复盘和迁移总结
```

建议每个薄弱点至少测试：

- 2 道同类新题；
- 1 道 CSP-S 历年真题；
- 1 道参数或条件变式题；
- 1 道跨模块迁移题。

判定标准：

| 真实盲测命中率 | 结论 | 系统动作 |
| ---: | --- | --- |
| ≥ 70% | 已确认薄弱点 | 高优先级训练 |
| 50%～70% | 中等可信 | 继续验证并训练 |
| 30%～50% | 弱假设 | 降低优先级 |
| < 30% | 误判或分类错误 | 重新分类 |
| 未测试 | 不允许下确定结论 | 安排诊断题 |

## 5. 套用到我的当前结果

## 5.1 部分分拆解和得分策略不足

当前证据：

- 相关历史题约 18 道；
- 多道题出现部分分停滞；
- 当前模拟验证命中率约 64%；
- DeepSeek 给出了暴力、特殊性质、分阶段优化等建议；
- CSP-S T3/T4 的确高度依赖部分分规划。

当前判断：

> 与 CSP-S 一等奖高度相关，现阶段可评为“中等可信的高价值弱点”，但还不能评为已确认。

仍需验证：

1. 选择 3～5 道未做过的 CSP-S T3/T4；
2. 做题前要求写 20/40/60/100 分方案；
3. 检查是否能在 20 分钟内找到暴力和特殊性质；
4. 检查部分分代码能否稳定通过对应子任务；
5. 比较实际可拿分与系统预计可拿分。

## 5.2 数学建模不足

当前问题：

- 证据题数量很多；
- `math` 分类覆盖过宽；
- “数学建模”“部分分”“溢出”“DP 状态”混在一起；
- 两个数学薄弱点标题高度重复；
- 很可能存在重复统计。

当前判断：

> 可能存在真实问题，但当前粒度不足，不能据此直接安排大量“数学题”。

需要先拆成：

- 数论与整除；
- 模运算和周期；
- 组合计数；
- 递推和数列；
- 式子推导；
- 不变量；
- 构造；
- 数据范围和溢出；
- 数学模型转 DP；
- 暴力到优化的复杂度推导。

之后分别计算历史证据、真题频率和盲测命中率。

## 5.3 区间 DP 不稳定

当前证据：

- 系统关联约 54 道题；
- 但其中疑似包含大量非典型区间 DP；
- 当前模拟验证 0/3 命中；
- 可能是二维数组和嵌套循环造成静态误判。

当前判断：

> 应降级为“待复核假设”，不能继续作为第一梯队训练重点。

下一步应该：

1. 人工或大模型重新审核 54 道题的题目类型；
2. 区分真正区间 DP、普通二维 DP、字符串区间处理和无关题；
3. 只保留题目语义和代码都支持区间 DP 的样本；
4. 用 3 道真正区间 DP 新题盲测；
5. 若仍不命中，则删除该主要弱点。

## 5.4 普通 DP 状态、转移和初始化问题

当前证据：

- 历史题较多；
- 有状态、初始化、边界和模型风险；
- 尚未完成真实新题验证；
- DeepSeek 覆盖只是部分问题簇。

当前判断：

> 是合理假设，但不能称为已确认短板。

需要进一步拆成：

- 不会定义状态；
- 状态正确但转移漏项；
- 初始化错误；
- 枚举顺序错误；
- 空间优化后破坏依赖；
- 数据类型或下标导致伪 DP 错误。

## 5.5 T1 稳定性

当前事实：

```text
csp_s_problem_map.json 中 T1_STABILITY 数量为 0
```

当前判断：

> 这是当前系统最大的结构性缺失之一。

因为一等奖训练首先要回答：

- T1 能否稳定接近满分？
- 是否会在简单题上因边界、读题或实现丢 30～100 分？
- T1 平均用时是多少？
- T1 一次提交通过率是多少？
- T1 出错后能否快速定位？

当前系统没有真实 T1 真题基准，因而无法可靠评估这些问题。

建议优先建立：

```text
2019—2025 T1 真题集合
每题知识点
每题实现风险
我的完成记录
首次提交结果
独立完成时间
错误类型
重做结果
```

## 5.6 T2 建模能力

当前系统把 100 道历史题放入 T2 建模桶，但这些不是 100 道 CSP-S T2 真题。

因此不能直接得出“T2 很强”。

需要验证：

- 近年真实 T2 是否能独立完成；
- 平均用时是否不超过合理预算；
- 建模后是否能一次或少量提交通过；
- 遇到不熟悉模型时能否拿部分分；
- 是否存在会算法但认不出模型的问题。

## 5.7 T3/T4 综合能力

当前系统有部分分分析，但缺少真实套题环境。

T3/T4 必须在四小时上下文中验证：

- 前两题消耗了多少时间；
- 开 T3/T4 时还剩多少时间；
- 是否先写暴力；
- 是否正确判断正解可行性；
- 是否及时止损；
- 最终拿到了哪些子任务；
- 是否因为调试一个错误浪费过多时间。

单题训练无法完全替代这一验证。

## 8. 建议的优先开发顺序

### 阶段一：建立真题事实库

1. 收集 2019—2025 CSP-S 提高级第二轮真题；
2. 保存年份、题号、位置和来源；
3. 标注知识点和部分分；
4. 对分类结果进行人工或双模型复核；
5. 为每个字段保存置信度和来源。

### 阶段二：建立知识能力树

1. 根据 NOI 大纲建立一级和二级知识点；
2. 根据竞赛训练需要拆出三级能力；
3. 增加非知识点能力；
4. 把现有算法模块映射到能力树；
5. 处理 `math`、`implementation` 等过宽分类。

### 阶段三：重新分类历史题

1. 用题面、标签、代码和提交轨迹共同判断；
2. 不能只靠代码正则；
3. 对高风险模块进行 DeepSeek 复核；
4. 合并重复弱点；
5. 删除明显误分类。

### 阶段四：建立预计失分模型

1. 统计真题频率；
2. 估计当前失败概率；
3. 估计可补救部分分；
4. 计算预计失分；
5. 按一等奖影响重新排序。

### 阶段五：真实盲测

1. 从未做真题中选择诊断题；
2. 记录真实过程；
3. 验证弱点；
4. 更新置信度；
5. 重新生成差距报告。

### 阶段六：真实套题验证

1. 至少完成 2 套初始基线模拟；
2. 后续每周至少 1 套；
3. 保存四题得分、时间和策略；
4. 连续多场达到安全线才允许提高 readiness；
5. 模拟数据不得混入真实成绩。

## 9. 建议的系统验收标准

### 9.1 数据验收

- 2019—2025 真题完整；
- 每年四题齐全；
- 每题有来源链接；
- 每题有知识点和部分分结构；

## 13. 最重要的产品原则

1. 先建立比赛目标基准，再评价个人历史。
2. 历史异常只能产生假设，不能自动产生定论。
3. 知识点必须细化到可训练、可验证的层级。
4. 弱点必须绑定 CSP-S 真题和预计失分。
5. 模拟数据不能冒充真实训练数据。
6. 看过题解的题不能作为盲测。
7. 系统必须能够否定自己先前的错误诊断。
8. 一等奖判断必须依赖真实套题表现。
9. 训练优先级由预计得分收益决定，不由历史题数单独决定。
10. 最终目的不是让报告更长，而是让下一道题、下一周训练和考场得分决策更准确。
