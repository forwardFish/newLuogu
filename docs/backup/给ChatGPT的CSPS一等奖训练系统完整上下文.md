
## 1. 用户目标

用户的明确目标是：

```text
为了 CSP-S 一等奖，进行有计划、有目的、有针对性的刷题训练。
```

用户并不想要泛泛而谈的建议，例如“多刷题、多总结、多模拟”。用户需要的是：

```text
基于历史代码和提交记录的诊断
基于诊断结果的题目安排
基于训练反馈的动态调整
面向 CSP-S 一等奖的阶段目标
明确到每天、每周、每个模块的训练计划
```

用户关心的问题包括：

```text
1. 当前 T1/T2/T3/T4 分数是怎么来的？
2. 已经做过的题目如何转化为能力画像？
3. 如何根据当前水平安排后续训练？
4. 如何针对性出题，而不是随机刷题？
5. 如何判断训练是否有效？
6. 如何尽可能提高 CSP-S 一等奖概率？
7. 能不能保证一定拿一等奖？
```

对最后一个问题，需要明确回答：

```text
不能承诺 100% 一定拿一等奖。
```

原因是 CSP-S 的最终结果受以下因素影响：

```text
比赛当天题目风格
所在省份或地区分数线
临场状态
时间分配
读题失误
代码低级错误
评测环境与数据强度
```

但是可以建立一套训练闭环，使用户逐步进入“稳定一等奖区间”，也就是：

```text
连续多场模拟赛达到或超过目标一等奖线
T1/T2 高正确率
T3/T4 有稳定部分分能力
失误率下降
调试时间下降
模块短板持续补齐
```

## 5. T1/T2/T3/T4 的定义

### 5.1 T1 稳定拿分

T1 衡量的是：

```text
简单题和中等偏简单题是否能稳定拿分
会做的题是否容易因为实现细节丢分
是否具备比赛开局快速拿分能力
```

对应 CSP-S 中：

```text
第 1 题
第 2 题偏基础部分
部分第 3 题的简单子任务
```

T1 关注的不是高深算法，而是：

```text
读题准确
边界处理
数组下标
初始化
数据范围
long long
多测清空
输出格式
模拟细节
简单贪心
基础 DP
```

### 5.2 T2 建模转化

T2 衡量的是：

```text
看到中档题后，能否识别算法方向
能否从题意转化为状态、图、判定函数或贪心结构
能否把常见算法模块稳定写出来
```

对应 CSP-S 中：

```text
第 2 题
第 3 题部分正解或较高部分分
部分第 4 题的可做子任务
```

T2 关注：

```text
DP 状态设计
图论建模
二分答案
贪心证明
搜索剪枝
树上基础问题
字符串处理
数学推导
```

### 5.3 T3 部分分策略

T3 衡量的是：

```text
面对难题时，不会满分正解也能拿部分分
是否会根据数据范围拆子任务
是否能写暴力、特殊性质、弱化版本
```

对应 CSP-S 中：

```text
第 3 题
第 4 题
所有不会正解但可以拿分的子任务
```

T3 的核心不是“能不能 AC”，而是：

```text
不会正解时能不能拿 30 分
能不能从 30 分扩展到 50 分
能不能利用特殊性质拿 70 分
能不能在有限时间内选择最划算的实现
```

### 5.4 T4 综合策略

T4 衡量的是：

```text
正式模拟赛中的综合表现
包括开题顺序、时间分配、调试节奏、弃题判断、部分分取舍
```

对应 CSP-S 整场比赛。

T4 关注：

```text
先做哪题
哪题不该硬刚
什么时候转向部分分
调试多久必须止损
是否能保住已会题的分数
是否能在最后时间补到额外分
```

### 6.2 第一版公式

T1：

```text
T1 稳定拿分
= 88
- 静态风险比例 * 24
- 卡题比例 * 18
+ AC 比例 * 10
```

解释：

```text
T1 题目通常不是算法最难，而是最怕实现错误。

如果 AC 比例较高，说明基础得分能力存在，会加分。
如果卡题比例高，说明简单/中档题也可能调试过久，会扣分。
```

```text
T2 建模转化
= 72
+ DP 覆盖加分
+ 图论覆盖加分
- 平均提交次数惩罚
```

解释：

```text
DP 和图论覆盖多，说明用户接触过较多中档题核心模型。
但如果平均提交次数偏高，说明从思路到代码的转化还不够稳定。
```

T3：

```text
T3 部分分策略
= 62
+ 数据结构覆盖加分
+ DP 覆盖加分
- 卡题比例惩罚
```

解释：

```text
T3/T4 常需要更复杂结构、DP 优化或数据结构。
但部分分策略不是只看高级算法，而是看是否能稳定从难题中拿到可拿分。
卡题多会明显影响这个分数。
```

T4：

```text
T4 综合策略
= 70
- 平均提交次数惩罚
- 卡题比例惩罚
+ AC 比例加分
```

解释：

```text
综合策略和调试成本高度相关。
平均提交次数高，说明比赛中可能花更多时间调试。
卡题多，说明止损和取舍能力需要训练。
AC 比例高，说明最终解决能力存在。
```

## 10. 后续训练计划生成原则

训练计划必须服务于 CSP-S 一等奖，而不是简单增加刷题数量。

### 10.1 训练比例建议

当前建议比例：

```text
T1 稳定性题：30%
T2 建模题：35%
T3 部分分难题：20%
T4 模拟赛与复盘：15%
```

解释：

```text
T1 不能弱，因为一等奖首先要稳住基础分。
T2 是当前优势，需要继续转化为稳定分。
T3 是当前短板，需要专项强化。
T4 用模拟赛检验所有能力。
```

### 10.2 每周训练结构

建议每周结构：

```text
周一：T1 稳定性 + 错题重写
周二：DP / 区间 DP / 背包 DP 专项
周三：图论 / 并查集 / 二分专项
周四：T3 部分分难题拆分训练
周五：字符串 / 数学 / 贪心专项
周六：CSP-S 四题模拟赛
周日：模拟赛复盘 + 高提交次数旧题复盘
```


## 14. 如何判断是否接近一等奖

不能用“刷了多少题”作为核心标准。应该用模拟赛表现判断。

建议达标线：

```text
T1/T2 正确率 >= 90%
T1 平均用时 <= 35 分钟
T2 平均用时 <= 60 分钟
T3 至少稳定拿 30 到 60 分
T4 至少能识别可拿分子任务
四题模拟赛总分 >= 目标省份一等奖线 + 20 分安全垫
连续 5 场模拟赛达到目标
```

如果达不到，应根据问题调整：

```text
T1 错：减少难题，增加稳定性训练
T2 错：补模块专项
T3 空：训练部分分拆解
T4 崩：增加模拟赛和策略复盘
调试太久：增加错题重写和代码模板规范
```
