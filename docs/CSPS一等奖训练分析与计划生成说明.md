# CSP-S 一等奖训练分析与计划生成说明

生成日期：2026-06-23

本文档说明当前本地洛谷代码分析系统如何对历史代码进行能力画像、如何计算 T1/T2/T3/T4 分数、如何根据已经做过的题目生成诊断结果，以及后续如何围绕 CSP-S 一等奖目标安排训练题目。

## 1. 目标定位

本系统的目的不是简单统计“刷了多少题”，而是建立一个训练闭环：

```text
历史提交与代码文件
-> 结构化分析
-> 能力画像
-> 薄弱点诊断
-> 针对性出题与训练计划
-> 新代码与新提交再次导入
-> 重新评估并调整计划
```

最终目标是尽可能提高 CSP-S 一等奖概率。

需要明确的是：系统不能承诺“百分百一定拿一等奖”。竞赛结果还受到当天题目风格、地区分数线、临场状态、时间分配、低级失误等因素影响。系统能做的是把训练变成可度量、可复盘、可调整的过程，让你逐步进入“稳定一等奖区间”。

## 3. T1/T2/T3/T4 的含义

当前系统把 CSP-S 训练能力拆成四类：

| 维度 | 含义 | 对应训练重点 |
| --- | --- | --- |
| T1 稳定拿分 | 简单题和中等偏简单题的稳定 AC 能力 | 实现细节、边界、模拟、简单贪心、基础 DP |
| T2 建模转化 | 中档题的建模、算法选择和转化能力 | DP、图论、二分、贪心证明、搜索 |
| T3 部分分策略 | 难题中分层拿分的能力 | 子任务、暴力、特殊性质、部分分代码 |
| T4 综合策略 | 模拟赛中的时间分配、取舍和调试能力 | 四题套卷、考试策略、复盘节奏 |

## 4. 当前打分逻辑

当前评分是第一阶段启发式模型，不是最终预测模型。它主要根据历史提交结果、代码复杂度、算法模块覆盖、卡题情况和静态风险进行估算。

### 4.1 参与评分的核心指标

提交表现：

```text
AC 题目数
未 AC 题目数
平均每题提交次数
多次失败后 AC 的题目数
长期未 AC 的题目数
部分分题目数
```

代码稳定性：

```text
数组边界风险
大全局数组
#define int long long
递归
潜在整型溢出
长代码
深层嵌套
复杂循环
```

算法模块覆盖：

```text
数学
字符串
图论
DP
贪心
搜索
并查集
二分
树
背包 DP
区间 DP
数据结构
```

### 4.2 第一版评分公式

当前系统中 T1/T2/T3/T4 的计算思路如下：

```text
T1 稳定拿分
= 88
- 静态风险比例 * 24
- 卡题比例 * 18
+ AC 比例 * 10
```

T1 主要衡量基础题和中档题能不能稳稳拿下。静态风险越高，越容易在简单题上因为边界、初始化、溢出、数组下标等问题丢分。

```text
T2 建模转化
= 72
+ DP 覆盖加分
+ 图论覆盖加分
- 平均提交次数惩罚
```

T2 主要衡量是否具备中档题的建模能力。DP、图论、搜索、二分、贪心等覆盖越多，说明题型经验越丰富；但如果平均提交次数过高，说明转化后实现或调试成本偏高。

```text
T3 部分分策略
= 62
+ 数据结构覆盖加分
+ DP 覆盖加分
- 卡题比例惩罚
```

T3 主要衡量面对难题时能否稳定拿部分分。CSP-S 中 T3/T4 不一定都要正解，但必须会根据数据范围和特殊性质设计 30/50/70 分做法。

```text
T4 综合策略
= 70
- 平均提交次数惩罚
- 卡题比例惩罚
+ AC 比例加分
```

T4 主要衡量模拟赛策略，包括做题顺序、是否硬刚、是否及时转向部分分、调试时间是否过长。

## 6. 如何根据已经做过的题生成分析

系统不是直接让模型阅读全部源码，而是先做结构化分析。

### 6.1 submission_manifest.json

作用：建立每份代码的基础档案。

记录字段包括：

```text
文件路径
题号
题目标题
提交记录 ID
提交结果
分数
语言
提交时间
文件大小
```

用途：

```text
知道每份代码对应哪道题
知道每道题做了多少次
知道哪些题 AC、部分分、未知或未完成
为后续训练计划提供历史基础
```

### 6.2 code_static_metrics.json

作用：分析每份代码的实现特征。

记录指标包括：

```text
代码行数
非空行数
最大行长
函数数量
循环数量
最大大括号嵌套深度
最大循环嵌套深度
是否使用 bits/stdc++.h
是否使用 #define int long long
是否使用全局数组
是否使用递归
是否使用 memset
是否可能整数溢出
是否可能数组边界风险
```

用途：

```text
发现实现习惯风险
筛选长代码和复杂代码
筛选适合深度复盘的代码
判断 T1/T2 稳定性
```

### 6.3 problem_code_timeline.json

作用：按题目聚合提交过程。

记录指标包括：

```text
每题提交次数
是否 AC
首次提交时间
最后提交时间
首次 AC 时间
最高分
最终结果

是否多次失败后 AC
是否长期未 AC
是否部分分
是否长代码题
是否深嵌套题
```

用途：

```text
识别高提交次数题
识别卡题
识别部分分题
识别值得复盘的题
分析调试成本
```

识别方向包括：

用途：

```text

```

## 7. 如何从分析结果生成后续训练计划

后续训练不应该随机刷题，而应该按能力缺口分配题目。

### 7.1 T1 稳定拿分训练

目标：

```text
减少会做但做错
减少低级错误
提高一次 AC 率
让 T1/T2 成为稳定得分来源
```

适合题型：

```text
模拟
枚举
前缀和
排序
简单贪心
简单 DP
基础字符串
基础图遍历
基础数学
```

训练方式：

```text
每天 3 到 5 道
限时完成
提交前必须检查
错题必须记录第一错误点
```

检查清单：

```text
数据范围是否会爆 int
数组是否开够
下标从 0 还是 1 开始
边界 n=1 是否处理
初始化是否完整
是否有多测清空
递归是否可能爆栈
输出格式是否正确
```

### 7.2 T2 建模转化训练

目标：

```text
提高中档题识别算法的速度
提高从题意到状态/图/判定函数的转化能力
```

适合题型：

```text
线性 DP
背包 DP
区间 DP
图论建模
最短路
并查集
二分答案
贪心证明
搜索剪枝
树上问题
```

训练方式：

```text
每 2 天一个模块小题组
每组 4 到 6 题
从基础到中档递进
每题必须写状态、转移、复杂度、边界
```

### 7.3 T3 部分分策略训练

目标：

```text
遇到不会正解的难题时，仍然能稳定拿 30 到 70 分
```

训练方式：

```text
每周 2 到 3 道难题
不要求一开始 AC
必须先拆分子任务
先写 30 分暴力
再写 50 分特殊性质
再尝试 70 分优化
最后学习正解
```

每道难题复盘模板：

```text
1. 满分正解是什么
2. 我能想到的暴力是什么
3. 哪些数据范围可以拿部分分
4. 哪些特殊性质可以利用
5. 如果比赛中只剩 40 分钟，我应该写哪个版本
6. 这题下次遇到同类题怎么判断
```

### 7.4 T4 综合策略训练

目标：

```text
训练正式比赛中的时间分配、取舍、调试和止损能力
```

训练方式：

```text
每周至少 1 次 4 题模拟
严格限时
按 CSP-S 节奏做
赛后必须复盘
```

模拟赛复盘模板：

```text
1. 开题顺序是否正确
2. 哪题花时间过长
3. 哪题应该先拿部分分
4. 哪题调试超过预期
5. 哪个错误最不应该出现
6. 如果重做一次，分数能提高多少
```

## 9. 建议训练周期


每天必须记录：

```text

模块

```

### 9.2 30 天冲刺计划

目标：建立 CSP-S 一等奖所需的稳定区间。

```text
第 1 周：实现稳定性 + 基础模块补强
第 2 周：DP、图论、二分、搜索专项
第 3 周：部分分策略 + 难题拆分训练
```

```text

```

如果某周模拟赛低于目标分，应立即调整：

```text
减少新题
增加复盘
重写错题
补对应模块基础题
```

## 10. 一等奖达标判断

不能只用刷题数量判断是否接近一等奖，应该用模拟赛表现判断。

建议达标线：

```text
T1/T2 正确率 >= 90%
T1 平均用时 <= 35 分钟
T2 平均用时 <= 60 分钟
T3 至少稳定拿 30 到 60 分
T4 至少能识别可拿分子任务
四题模拟赛总分 >= 目标省份一等奖线 + 20 分安全垫
连续 5 场模拟赛达到目标
```

只有连续模拟赛达标，才说明接近“稳定一等奖区间”。
